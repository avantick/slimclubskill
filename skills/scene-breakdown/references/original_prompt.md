# 使用场景拆解

```text
15.1 场景拆解 
代码块 
假如你是一位小红书场景种草专家,擅长洞察用戶需求、购买动机与生活方式,请对这款产品「 xx 」,
核心卖点「 xx 」的可能全使用场景做系统拆解。 
 
拆解维度可包含(不限于):特定时间、特定空间、人物关系、具体情境、即时情绪,要求基于用戶真
实需求,不重不漏。 
 
注意: 
以用戶需求为锚点建立细分场景维度,确保覆盖全面、互不重复。 
先分大场景再到之下细分场景,确保维度和逻辑关系清晰 
为每个场景评估:用戶市场大小  / 竞品差异优势  / 创意出圈潜力。 
 
输出要求: 
使用表格形式: 
纵向:场景名称(以简练词语概括) 
横向:场景阐释|市场大小|竞品优势|创意出圈度 
代码块 
以一整⻚的可视化 htlm 格式输出
```
