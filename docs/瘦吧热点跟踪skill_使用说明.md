# 瘦吧热点跟踪skill 使用说明

## 一、Skill 定位

**瘦吧热点跟踪skill** 只负责一件事：

> 从 UApiPro 获取小红书热榜，并输出中立、原始、可追溯的热点列表。

它是整个链路的第一步，只做“热点跟踪”，不做业务判断。

---

## 二、它适合解决什么问题？

适合用于：

- 获取当前小红书热榜。
- 定期记录 UApiPro 小红书热榜变化。
- 为后续产品匹配和选题卡生成提供原始热点输入。
- 培训演示“热点监测不带业务倾向”的第一步。

---

## 三、它不做什么？

这个 Skill 不做：

- 不判断热点是否适合瘦吧。
- 不分类变瘦、变美、变健康。
- 不推断用户痛点。
- 不匹配产品。
- 不生成标题或文案。
- 不输出选题卡。
- 不判断是否值得跟进。

这些动作留给后续的 **瘦吧小红书选题卡skill** 或内容智能体完成。

---

## 四、数据来源

默认数据源：

```text
https://uapis.cn/hotboard/xiaohongshu
```

注意：

- 这是 UApiPro 的公开小红书热榜聚合页。
- 它不是小红书官方 API。
- 输出时需要标注数据源和抓取时间。

---

## 五、推荐调用方式

### 方式 1：获取当前热榜

```text
请使用“瘦吧热点跟踪skill”，从 UApiPro 获取当前小红书热榜。
只输出原始热榜列表，不要做产品匹配、用户痛点推断和机会判断。
```

### 方式 2：生成热点跟踪报告

```text
请使用“瘦吧热点跟踪skill”，生成一份当前 UApiPro 小红书热榜跟踪报告。
需要包含：
1. 数据源说明
2. 抓取时间
3. 页面更新时间
4. 排名、热点标题、热度值、原始链接
5. 数据边界说明
6. 下游交接说明
```

### 方式 3：为后续选题卡准备输入

```text
请使用“瘦吧热点跟踪skill”，获取 UApiPro 小红书热榜，并整理成可以交给“瘦吧小红书选题卡skill”的表格。
```

---

## 六、标准输出字段

| 字段 | 说明 |
|---|---|
| fetched_at | 抓取时间 |
| source | UApiPro 小红书热榜公开聚合页 |
| source_url | 数据源 URL |
| board_updated_at | 页面显示的更新时间 |
| rank | 热榜排名 |
| title | 热点标题 |
| heat | 热度值 |
| original_url | 原始链接 |
| evidence_status | 证据状态 |

## 八、与后续 Skill 的关系

```text
瘦吧热点跟踪skill
↓
获取中立热榜
↓
输入产品库
↓
瘦吧小红书选题卡skill
↓
生成选题卡
↓
小红书内容智能体
↓
生成文案
```

一句话总结：

> 热点跟踪只负责把真实热点拿回来，不负责判断热点适不适合产品。
